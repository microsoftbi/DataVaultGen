## 元数据导入流程设计（待确认）

### 当前流程（4步）

```
Step 1: 选择源连接
Step 2: 选择数据表（多选+批量导入）
Step 3: 预览字段 + 直接导入到 META
Step 4: 导入完成
```

### 新流程（5步）

```
Step 1: 选择源连接
Step 2: 选择数据表（单选，一次只处理一张表）

Step 3 (NEW): 选择要导入的列
  ┌─────────────────────────────────────────┐
  │  当前表: [dbo].[CUSTOMER]  共 18 个列   │
  │                                         │
  │  □ 列名      类型      长度  角色推荐    │
  │  ☑ ID        INT       -     BK ✓     │
  │  ☑ NAME      NVARCHAR  100   DI        │
  │  ☑ EMAIL     NVARCHAR  255   DI        │
  │  ☐ LOAD_DTS  DATETIME  -     [技术字段] │ (默认不勾选)
  │  ☐ HK        CHAR      32    [技术字段] │ (默认不勾选)
  │  ...                                     │
  │                                         │
  │  [全选] [清除技术字段] [恢复默认]        │
  └─────────────────────────────────────────┘
                       ↓
Step 4: 确认导入
  ┌─────────────────────────────────────────┐
  │  表: dbo.CUSTOMER                       │
  │  将导入: 15 个字段                       │
  │  将跳过: 3 个技术字段                    │
  │                                         │
  │  [上一步]  [确认导入 →]                  │
  └─────────────────────────────────────────┘

Step 5: 导入完成（同现步骤4）
```

### 关键变化

| 项目 | 当前 | 新方案 |
|------|------|--------|
| 列选择 | 无选择，全部导入 | 每列可勾选/取消 |
| 技术字段 | 后端自动跳过，不可见 | 前端灰掉/默认不勾选，用户可见 |
| 批量导入 | 一次选多张表直接导入 | 一次只处理一张表 |
| 角色推荐 | 导入后自动推荐 BK/DI | 导入前在列旁显示推荐标记 |

### 后端改动

`POST /api/meta/import` 接受可选参数 `columns: list[str]`，只导入指定的列。不传则全部导入。

### 前端改动

- Step 3 新增列选择表格（checkbox + 智能默认值）
- 增加"全选"、"清除技术字段"按钮
- 移除批量导入功能（改为一张一张导入更可控）
- Step 4 改为确认页，显示导入摘要
